# Pitfall Journal

> 核心：每个踩坑都是未来的约束规则。闭环：踩坑 → 规则 → Skill。

## 触发条件

- AI 生成了错误代码且被 io-wy 纠正
- AI 遗漏了边界条件导致 bug
- AI 使用了不存在的 API（幻觉）
- AI 改不全——局部修改但遗漏了全局影响
- AI 在不确定时编造了信息
- io-wy 说「踩坑」「记录 PIT」「pitfall」

## 记录流程

每次犯错后立即执行，不可跳过：

### Step 1：记录 PIT

在本文件「踩坑记录」区域追加一条 PIT，使用下方格式。

### Step 2：判断进化条件

检查同类型 PIT 是否已出现 >= 2 次：
- **首次**：仅记录，状态标记「新增」
- **重复 2+ 次**：提炼为 CLAUDE.md 编码约束，更新状态为「已提炼规则」，在 PIT 中标注约束编号（如 C-11）
- **规则验证 2+ 次**：评估是否需要固化为独立 Skill，更新状态为「已固化 Skill」
- **规则效果有限**：标注「通用知识可覆盖」

### Step 3：告知 io-wy

输出一行：
```
PIT-xxx 已记录（类型/严重度/状态）。同类型累计 N 次。
```

如果达到进化条件，额外告知：
```
同类 PIT 已达 2+ 次，建议提炼为 CLAUDE.md 约束。
```

## 记录格式

```markdown
### PIT-XXX: <简短标题>

- **日期**: YYYY-MM-DD
- **类型**: 改不全 | 幻觉 | 环境盲区 | 模式匹配代替验证 | 不会说不知道 | 并发 | 资源泄漏 | 错误处理缺失 | 协议漂移 | review-false-positive
- **严重度**: Critical | High | Medium
- **场景**: 做什么任务时发生
- **现象**: 出了什么问题
- **根因**: 为什么（5 Why）
- **修复**: 怎么修的
- **规则提炼**: 是否需要新增 CLAUDE.md 约束？
- **状态**: 新增 | 已提炼规则 | 已固化 Skill | 通用知识可覆盖
```

## 10 种 PIT 类型说明

| 类型 | 典型表现 | 严重度基准 |
|------|---------|-----------|
| **改不全** | 改了函数签名但遗漏调用点；加了字段但没加 migration | High |
| **幻觉** | 使用了不存在的 API、编造了函数签名 | Critical |
| **环境盲区** | 不知道项目的 Go 版本、配置、已有依赖 | Medium |
| **模式匹配代替验证** | 凭训练记忆断言行为，未用工具查证 | High |
| **不会说不知道** | 不确定时编造信息而非诚实告知 | Critical |
| **并发** | 共享状态无保护、锁粒度不当 | High |
| **资源泄漏** | Open 无 Close、goroutine 泄漏 | High |
| **错误处理缺失** | 吞错误 `_ =`、错误无上下文 | Medium |
| **协议漂移** | 实现与参考文档不一致 | High |
| **review-false-positive** | 审查误报率过高 | Medium |

## 5 Why 根因分析示例

```
PIT-005: 用户注册接口返回 500

1 Why: 数据库写入失败
2 Why: email 字段长度超限
3 Why: 未校验用户输入长度
4 Why: CLAUDE.md 无输入长度校验约束
5 Why: C-05 只说了"必须校验"但没说校验什么

→ 提炼规则：C-05 补充"所有 string 输入必须校验长度上限"
```

## 进化判断

| 重复次数 | 动作 |
|----------|------|
| 1 次 | 仅记录 PIT |
| 2+ 次 | 提炼规则，加入 CLAUDE.md 编码约束 |
| 规则验证 2+ 次 | 固化为独立 Skill |
| 规则效果有限 | 标注「通用知识可覆盖」 |

## 常见错误

| 错误 | 后果 | 预防 |
|------|------|------|
| 犯错后不记录 | 同类错误反复出现 | Step 1 不可跳过 |
| 记录太模糊（"代码有 bug"） | 无法提炼规则 | 每条 PIT 必须有类型和根因 |
| 只记录不进化 | PIT 堆积，知识不流动 | Step 2 每次检查进化条件 |
| 根因只写表面原因 | 同根因不同表现反复出现 | 5 Why 至少挖到第 3 层 |
| 进化后不更新 PIT 状态 | PIT 和规则重复存在 | 进化后立即更新状态 |

## 验证清单

- [ ] 犯错后 PIT 已记录
- [ ] PIT 包含完整的类型/严重度/根因
- [ ] 同类型 PIT >= 2 次已提炼规则
- [ ] 已提炼规则的 PIT 状态已更新
- [ ] 进化后已通知 io-wy
