# Knowledge Loop

> 核心：知识不是静态的文档，是活的系统。从错误中来，到约束中去，最终消失在自动化里。

## 触发条件

- PIT 记录新增后（评估是否需要进化）
- 定期回顾（建议每月一次）
- io-wy 说「知识闭环」「knowledge loop」「进化」「evolve」「知识管理」
- 想要理解项目约束体系是如何演化的

## 五级进化模型

```
Level 1: 踩坑记录 (PIT)     — 发生了什么？
    ↓ 重复 2+ 次
Level 2: 约束规则 (Rule)    — 不能这么做！
    ↓ 验证有效 2+ 次
Level 3: 专业 Skill         — 完整工作流 + 检查清单
    ↓ 稳定运行 5+ 次
Level 4: 自动化 (Command/MCP) — 按个按钮就行
    ↓ 使用者无感
Level 5: 隐形基础设施        — 系统自动执行，开发者不需要知道
```

### Level 1 → Level 2：踩坑 → 规则

**判断标准**：
- 同类型 PIT 出现 >= 2 次
- 根因分析指向同一个系统性缺陷

**执行动作**：
1. 提炼规则（反例免疫格式：WRONG + CORRECT + Why）
2. 写入 `CLAUDE.md` 编码约束（C-xx）
3. 更新 PIT 状态为「已提炼规则」
4. 在 PIT 记录中标注对应的约束编号

**示例**：
```
PIT-001: 吞错误导致线上故障排查 2 小时
PIT-003: 吞错误导致配置加载失败静默降级
→ 提炼为 C-01：错误处理——禁止吞错误
```

### Level 2 → Level 3：规则 → Skill

**判断标准**：
- 规则在 >= 2 个不同场景下被验证有效
- 单条规则不够，需要完整的工作流指导

**执行动作**：
1. 创建独立 Skill 文件
2. 包含：触发条件 + 完整工作流 + 反例免疫约束 + 验证清单
3. 更新 PIT 状态为「已固化 Skill」
4. 更新 CLAUDE.md L4 层表格

**示例**：
```
C-10: 协议实现必须对照参考文档
+ 多次协议实现踩坑经验
→ 固化为 protocol-driven-development Skill
```

### Level 3 → Level 4：Skill → 自动化

**判断标准**：
- Skill 稳定运行 >= 5 次，无重大调整
- 工作流步骤可以程序化执行
- 人工决策点 <= 3 个

**执行动作**：
1. 将 Skill 核心逻辑封装为 Command 或 MCP 工具
2. 保留 Skill 文件作为文档
3. Command 做薄壳路由，Skill 做业务逻辑

**示例**：
```
change-impact-scan Skill
→ 封装为 MCP 工具，AI 调用工具自动扫描
→ 开发者不需要手动执行 Grep 步骤
```

### Level 4 → Level 5：自动化 → 隐形

**判断标准**：
- 工具被无缝集成到日常流程中
- 开发者不再需要显式触发
- 失败时才需要人工介入

**执行动作**：
1. 集成到 pre-commit hook / CI pipeline / IDE 插件
2. 失败时自动提示修复建议
3. 开发者只需处理异常情况

**示例**：
```
lint 规则 → golangci-lint 自动运行
错误处理约束 → pre-commit hook 检测 _ := ... 模式
```

## 退化检测

知识也会过时。定期检查：

| 检查项 | 频率 | 判定标准 | 动作 |
|--------|------|----------|------|
| PIT 是否仍然相关 | 每季度 | 关联代码是否已重构/删除 | 过时则标记「已归档」 |
| 约束规则是否仍有效 | 每季度 | 最近 3 个月是否触发过 | 未触发则降级为注释 |
| Skill 是否仍被使用 | 每月 | 触发次数 >= 1 | 未触发则评估是否需要 |
| 自动化工具是否正常 | 每周 | CI 通过率 > 95% | 失败则修复或回退 |

## 执行方式

### 日常操作（AI 自动执行）
- 犯错时自动记录 PIT
- 每次记录后检查是否达到进化条件
- 达到条件时提示 io-wy

### 定期回顾（和 io-wy 一起执行）
1. 读取 pitfall-journal.md，统计 PIT 状态分布
2. 识别待进化的 PIT / 待降级的规则
3. 执行进化或降级
4. 输出知识健康度报告

## 常见错误

| 错误 | 后果 | 预防 |
|------|------|------|
| 只记录不进化 | PIT 堆积，知识不流动 | 每次记录后检查进化条件 |
| 跳级进化（PIT 直接自动化） | 自动化了错误的东西 | 严格按级进化，每级有验证 |
| 只升不降 | 过时规则成为负担 | 定期退化检测 |
| 进化后不清理 | PIT 和规则重复存在 | 进化后更新 PIT 状态 |

- [ ] 过时知识已标记归档
