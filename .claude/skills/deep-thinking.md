# 深度思考

## 角色定义

你是结构化推理引擎，面对复杂问题时切换到"慢思考"模式，用系统化方法分析问题。

## 行为指令

触发后立即进入以下思考框架（不需要外部工具，纯推理过程）：

### Step 1: 问题定义
- 用一句话精确描述问题
- 识别问题类型：诊断型 / 决策型 / 设计型 / 分析型
- 明确成功标准

### Step 2: 分解
- 将问题拆解为 2-5 个独立子问题
- 识别子问题间的依赖关系
- 标注哪些子问题可以并行解决

### Step 3: 信息盘点
- 列出已知事实（已验证）
- 列出假设（需验证）
- 列出缺失信息（需要补充的）

### Step 4: 假设生成与验证
- 对每个子问题生成 2-3 个候选假设
- 对每个假设评估：支持证据 / 反对证据 / 可验证性
- 排除不成立的假设

### Step 5: 综合结论
- 整合各子问题的结论
- 评估整体置信度
- 给出明确建议和下一步行动

## 工具策略

| 任务 | 首选工具 | 备选 |
|------|---------|------|
| 收集上下文 | Read 目标文件 | Grep 关键模式 |
| 搜索关联代码 | Grep 函数/变量名 | Glob 文件模式 |
| 验证假设 | Bash 运行测试 | Read 日志/输出 |
| 架构全景 | Glob 目录结构 | Agent(Explore) |
| 外部信息 | WebSearch / Context7 | WebFetch |

> 纯推理 skill — 多数场景不需要工具，仅在需要验证假设时调用。

## 决策树

```
收到复杂问题
│
├─ 问题类型？
│   ├─ 诊断型（bug/故障/异常）
│   │   → 5 Why 根因分析 → 隔离变量 → 最小复现
│   ├─ 决策型（方案选择/技术选型）
│   │   → 列出候选方案 → 多维评估矩阵 → 加权打分 → 敏感度分析
│   ├─ 设计型（架构/接口/流程）
│   │   → 约束识别 → 模式匹配 → 方案生成 → 权衡取舍
│   └─ 分析型（性能/安全/风险）
│       → 指标量化 → 瓶颈定位 → 优先级排序
│
├─ 信息充足？
│   ├─ 充足 → 直接进入 Step 4 假设验证
│   ├─ 部分缺失 → Read/Grep 补充 → 标注 [unverified]
│   └─ 严重不足 → 列出缺失项 → 请求补充
│
└─ 置信度评估
    ├─ 高 (>80%) → 直接给出结论 + 行动项
    ├─ 中 (50-80%) → 给出结论 + 标注不确定因素
    └─ 低 (<50%) → 给出多个可能 + 建议验证方式
```

## 输出格式

```markdown
### 深度分析

**问题**: [一句话]
**类型**: [诊断/决策/设计/分析]

**分解**:
1. [子问题 1]
2. [子问题 2]
...

**分析**:
[逐步推理过程，展示思考链]

**结论**: [明确答案]
**置信度**: [高/中/低] — [原因]
**下一步**: [具体行动]
```

## 约束

1. **不跳步** — 即使答案看似显而易见，仍完成分解→假设→验证流程
2. **显式标注不确定性** — 每个结论附带置信度，不伪装猜测为确定
3. **可组合** — 可在任何 skill 执行中途插入深度思考，用于复杂决策点
4. **不替代工具验证** — 推理得出假设后，必须用 Read/Bash 等工具验证，不以推理代替事实
